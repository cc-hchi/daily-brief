# 项目现状与最终目标（理解记录）

## 一、项目现状（基于已读取文件）
1. **核心产出**：
   - `daily_brief/` 目录下生成静态网页（`index.html` + `archive/`），用于每日信息摘要展示。
   - 使用 `template.html` + `data.json` 进行渲染；`build_daily.py` 负责生成。

2. **数据来源与采集**：
   - `skills/local-slack-scripts/` 提供一组本地 Python 脚本，用于从 Slack 拉取频道/私信内容、扫描未处理消息、下载附件等。
   - `downloads/summary.json` 保存历史拉取的数据样本。

3. **偏好与画像**：
   - `memory/news_preferences.md` 与 `memory/news_profile.json` 记录了内容偏好与关注方向。

4. **部署形态**：
   - `daily_brief/.github/workflows/deploy.yml` 显示当前存在 GitHub Actions 的部署流程。

5. **输出样式**：
   - `daily_brief/archive/` 存在按日期归档的 HTML 页面，说明存在每日生成与历史留存。

---

## 二、项目最终目标（基于用户描述）
- 这是一个**单人使用**的个人网站。
- 目标是提供**个性化信息获取**与**内容服务**，服务对象仅为用户自己。
- 核心价值不是信息量，而是“**有用、可行动、能持续积累的个人信息服务**”。
- **网页最终呈现方式为中文**。
- **达到世界最高、最先进、最专业的水准（高级）**。
- **验收标准**：由助手使用 Web Browser 能力，亲自查看浏览器渲染后的网页，判断是否符合预期。

---

## 三、现状与目标的关键对齐点
- 已具备“每日生成 + 归档”的基础形态。
- 已具备“从 Slack 抽取数据 + 模板化呈现”的采集与展示链路。
- 已具备“偏好文件”用于个性化方向，但尚未形成闭环或交互反馈。
